# CoastGuesser WebRTC Referee Duel

별도 백엔드 없이 WebRTC DataChannel로 연결하는 1P vs 2P 대전 MVP입니다.

## 구조

- Host: 방장/심판/관전자 역할입니다.
- 1P, 2P: 실제 추측을 제출하는 플레이어입니다.
- 연결 형태: Host를 중심으로 1P와 2P가 각각 직접 연결되는 star topology입니다.
- 라운드: 10라운드
- 제한 시간: 라운드당 30초
- 점수 계산: Host 브라우저가 정답과 플레이어 추측 거리로 계산합니다.
- 문제 공유: MVP 단계라 라운드 좌표를 플레이어에게 공유하고 각 클라이언트가 직접 렌더링합니다.
- 연결 코드: SDP를 gzip 압축한 `G.` 형식 코드로 교환합니다. 압축 API가 없는 브라우저에서는 `J.` 형식으로 자동 대체됩니다.

## 실행

프로젝트의 `webrtc-referee` 폴더에서 로컬 서버를 실행합니다.

```powershell
python -m http.server 8000
```

브라우저에서 엽니다.

```text
http://localhost:8000
```

## 연결 순서

1. 방장 브라우저에서 `방장으로 열기`를 누릅니다.
2. 1P 브라우저에서 `참가자로 연결 준비`를 누르고 생성된 Offer를 방장에게 전달합니다.
3. 방장이 1P Offer를 붙여넣고 `1P Answer 만들기`를 누릅니다.
4. 1P가 방장의 Answer를 붙여넣고 `Answer 적용`을 누릅니다.
5. 2P도 같은 방식으로 연결합니다.
6. 1P와 2P가 Ready를 누르면 방장이 `10라운드 시작`을 누릅니다.

게임이 시작되면 Offer/Answer 입력 패널은 자동으로 숨겨집니다.
대전이 끝난 뒤에는 방장 화면의 `새 10라운드 시작` 버튼으로 같은 WebRTC 연결을 유지한 채 다음 매치를 바로 시작할 수 있습니다.

## 결과 표시

- 1P 추측: 주황색
- 2P 추측: 파란색
- 정답: 청록색
- 최근 라운드 결과판에서 각 플레이어의 이번 라운드 획득 점수와 거리를 확인할 수 있습니다.

## 주의

- 완전한 치팅 방지용 구조는 아닙니다. 친구끼리 즐기는 캐주얼 P2P 대전 MVP입니다.
- 외부 네트워크에서는 NAT 환경에 따라 WebRTC 연결이 실패할 수 있습니다.
- 해안선 데이터는 GitHub 원본 raw URL에서 불러옵니다.
